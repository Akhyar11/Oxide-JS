## Key Changes in this Version

- **Neural Memory Management (Bounded Workspace Policy)**: Refactored all layers and optimizers to utilize persistent, pre-allocated buffers via `MemoryManager.ensureCapacity`. This eliminates GC overhead during long training loops and supports variable sequence lengths without reallocation.
- **Native GRU Implementation**: Finalized native backend support for the GRU family with `forward` and `backward` operations in Rust.
- **Full Recurrent Batching Support**: All recurrent layers (RNN, LSTM, GRU) now support native batched processing for significant throughput gains.
- **Zero-Allocation Cost Functions**: Refactored cost functions to support optional pre-allocated output buffers for gradient calculation.

## Interpretation

- The Bounded Workspace Policy has stabilized memory usage, preventing RAM growth during training.
- GRU performance is now on par with LSTM despite its gating complexity, thanks to optimized native implementation.
- High throughput across all recurrent families allows for efficient training of sequence models on CPU.
