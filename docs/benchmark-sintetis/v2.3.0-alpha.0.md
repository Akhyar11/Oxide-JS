## Perubahan Penting Dibanding Versi Sebelumnya

- **Native GRU Implementation**: Menyelesaikan dukungan native backend untuk seluruh model recurrent utama dengan mengimplementasikan `gru_forward_native_into` dan `gru_backward_native_into` di Rust.
- **Full Recurrent Batching Support**: Memastikan seluruh layer recurrent (RNN, LSTM, GRU) mendukung pemrosesan batched secara native untuk akselerasi performa yang signifikan dibandingkan fallback JavaScript.

## Interpretasi

- GRU memiliki overhead performa antara RNN dan LSTM, yang sesuai prediksinya mengingat kompleksitas gating-nya.
- Native backend memberikan throughput yang sangat tinggi untuk seluruh keluarga recurrent, memungkinkan training model sequence yang lebih besar pada hardware CPU.

## Tindak Lanjut

- Optimasi lebih lanjut pada memory sharing antara Node.js dan Rust untuk mengurangi overhead penyalinan data pada sequence yang sangat panjang.
